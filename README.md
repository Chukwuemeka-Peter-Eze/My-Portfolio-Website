## Overview

This portfolio serves as a live demonstration of the engineering discipline I bring to cloud and DevOps roles. The same principles I apply when designing AWS infrastructure; separation of concerns, type safety, automation, maintainability, and security, are applied here in the codebase.

**What makes this portfolio different from a template:**

- All content lives in typed TypeScript data files. Adding a project, certification, or blog article means editing one file, no touching components
- Live GitHub API integration pulls real repository data, profile stats, and activity, not placeholder text
- A working contact form with a server-side API route keeps the email API key off the client
- A dedicated Certification Journey page documents the full learning roadmap with progress bars, skill pills, and related projects
- Every section responds correctly to dark and light mode; tested and verified

## Features

### Core Experience
- Dark and light mode with system preference detection and smooth CSS transitions
- Fully responsive layout across mobile, tablet, and desktop viewports
- Animated section entrances using Framer Motion (FadeIn, FadeUp, ScaleIn, Stagger, StaggerItem)
- Active section highlighting in the navbar using IntersectionObserver API
- Scroll-aware sticky navbar with backdrop blur and shadow on scroll
- Back-to-top button with smooth scroll behavior
- Mobile hamburger menu with full navigation parity

### SEO and Discoverability
- Open Graph metadata for LinkedIn and social media previews
- Twitter Card metadata
- Keyword metadata targeting Cloud Security, DevOps, AWS, Kubernetes, Terraform
- Sitemap.xml auto-generated by Next.js metadata API
- Robots.txt auto-generated by Next.js metadata API
- Canonical URL configuration

### GitHub API Integration
- Live GitHub profile stats: public repositories, followers, member since year
- Live repository feed showing 6 most recently updated repositories
- Real-time language detection, star count, fork count, and last updated date per repo
- Animated loading skeleton while data is being fetched
- Graceful error fallback if the GitHub API is rate-limited or unavailable

### Contact System
- Working contact form powered by the Resend email API
- Server-side API route at `/api/contact` - API key never exposed to the client
- Client-side form validation with required field enforcement
- Loading state on submit button to prevent double submission
- Success and error alerts with clear user feedback
- Messages delivered directly to ProtonMail inbox

### Calendly Integration
- Book a Meeting button in both the Hero section and Contact section
- Direct link to Calendly scheduling page for immediate recruiter action

### Performance
- Next.js Image component on the profile photo with `priority` and `loading="eager"` to optimize LCP
- Font optimization via `next/font/google` - no layout shift
- Component-level code splitting via Next.js App Router

## Getting Started

### Prerequisites

- Node.js 18 or higher
- npm or yarn
- A free Resend account for the contact form email functionality

### Installation

```bash
# Clone the repository
git clone https://github.com/Chukwuemeka-Peter-Eze/cloud-portfolio.git

# Navigate into the project directory
cd cloud-portfolio

# Install all dependencies
npm install

# Start the local development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser to view the site.

### Available Scripts

```bash
npm run dev      # Start development server with hot reload
npm run build    # Build for production
npm run start    # Start production server locally
npm run lint     # Run ESLint
```

## Environment Variables

Create a `.env.local` file in the root of the project and add the following:

```env
RESEND_API_KEY=your_resend_api_key_here
```

The contact form will not send emails without this key. You can obtain a free API key at [resend.com](https://resend.com). The free tier supports 3,000 emails per month.

> **Security note:** Never commit `.env.local` to version control. It is already included in `.gitignore`. The API key is only used server-side inside `/app/api/contact/route.ts` and is never exposed to the browser.

## Deployment

This portfolio is deployed on **AWS Amplify** with automatic continuous deployment triggered on every push to the main branch.

### AWS Amplify Setup

1. Log into the [AWS Console](https://console.aws.amazon.com)
2. Open **AWS Amplify** and click **New App → Host Web App**
3. Select **GitHub** as the source and authorize AWS Amplify
4. Choose the `cloud-portfolio` repository and the `main` branch
5. Amplify automatically detects Next.js and configures the build settings
6. Under **Environment Variables**, add `RESEND_API_KEY` with your Resend API key
7. Click **Save and Deploy**

Amplify will pull the code, build the Next.js application, and deploy it. A temporary URL is provided immediately (e.g., `https://main.abcd1234.amplifyapp.com`).

### Custom Domain Setup

The domain `chukwuemekapetereze.online` is registered on Namecheap and connected to AWS Amplify:

1. In Amplify, go to **Domain Management → Add Domain**
2. Enter `chukwuemekapetereze.online`
3. Copy the CNAME records provided by Amplify
4. In Namecheap DNS settings, add the CNAME records
5. Wait for DNS propagation (up to 48 hours)
6. Amplify automatically provisions an SSL/TLS certificate via AWS Certificate Manager

### Build Configuration

```bash
# Build command
npm run build

# Output directory
.next/
```

## Engineering Highlights

This portfolio demonstrates software engineering practices that transfer directly to cloud and DevOps roles:

**Separation of concerns** - data, UI components, section components, hooks, and TypeScript types each live in their own dedicated layer. Adding a new project, certification, or blog article requires editing exactly one data file with no component changes required.

**TypeScript throughout** - every component, hook, API route, and data file is fully typed with explicit interfaces. Shared types are centralized in `src/types/index.ts` to prevent duplication.

**Live API integration** - the GitHub section fetches real data from the GitHub REST API using `Promise.all` for parallel requests, with proper loading states, error boundaries, and graceful degradation when the API is unavailable.

**Server-side API security** - the contact form submits to a Next.js server-side route (`/api/contact`) which calls the Resend API. The `RESEND_API_KEY` environment variable is only accessible server-side and never reaches the browser.

**Theme system** - dark and light mode are implemented using `next-themes` with Tailwind's `dark:` variant applied consistently across every component. Theme changes apply instantly without a page reload or flash of unstyled content.

**Reusable animation system** - five Framer Motion wrapper components (FadeIn, FadeUp, ScaleIn, Stagger, StaggerItem) cover every animation pattern on the site. Animations trigger once on scroll into view using `whileInView` with `viewport={{ once: true }}`.

**SEO architecture** - Open Graph, Twitter Card, author metadata, keywords, canonical URL, sitemap.xml, and robots.txt are all configured using the Next.js App Router Metadata API. The sitemap and robots files are programmatically generated at build time.

**Performance** - the Largest Contentful Paint (LCP) image uses `priority` and `loading="eager"` per Next.js recommendations. Fonts are loaded via `next/font/google` with no layout shift. Each page section is a separate component enabling automatic code splitting.

**Component architecture** - the `ui/` folder contains primitive components (Button, Card, Container, Section, Badge, Heading) that sections compose together. This mirrors the atomic design pattern used in production design systems.
